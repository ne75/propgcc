\subsection{Detailed Description}
Provides A\+PI for implementation of pthread functions. 

P\+O\+S\+IX threads are a set of functions that support applications with requirements for multiple flows of control, called threads, within a process. Multithreading is used to improve the performance of a program.

The pthread module provides a subset of the P\+O\+S\+IX standard pthread library for allowing multiple threads to run on multiple C\+OG instances of the Propeller\+G\+CC L\+MM interpreter. Multiple threads can also run on an X\+MM interpreter, but only one X\+MM interpreter can run at any given time (this restriction will probably be lifted soon).

\begin{DoxyPrecond}{Precondition}
It is very important to understand that pthreads provide a cooperative non-\/preemptive multithreading system. The consequences are that certain rules will apply. I.\+E. \begin{DoxyItemize}
\item The Propeller \mbox{\hyperlink{propeller_8h_abfb115793a1f26315b713a46891d53b2}{waitcnt()}} function can not be used. \item Delays can only be introduced with \mbox{\hyperlink{sys_2unistd_8h_ac0383719bafa9b5b2aa25fabc586bd1f}{usleep()}} and its variations. \item Standard \mbox{\hyperlink{stdio_8h_a55f3d956bb2ba20b5059e9e25484a47f}{printf()}} and friends must be used. \item Do not use -\/Dprintf=\+\_\+\+\_\+simple\+\_\+printf with pthreads. \item Threads must not hog the L\+MM C\+OG kernel; they must yield. \item Gobal data must be protected by mutex/semaphore.\end{DoxyItemize}
